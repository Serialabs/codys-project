# Open Questions & Verification Steps

Despite extensive research, several aspects of KNO Post‑Purchase Surveys remain unclear or undocumented.  The following questions should be clarified before finalising the MVP or planning subsequent versions.  Suggested verification steps are included.

| Question | Why it matters | How to verify |
| --- | --- | --- |
| **How are “Automated Insights” and benchmarking calculated?** | Marketing copy mentions demographic/psychographic analysis and benchmarks against other brands【624157833394015†L40-L87】, but no public documentation explains the methodology. | Reach out to KNO support or request a demo to understand what data they collect (age, gender, social networks) and whether third‑party data is used. Verify if this requires user consent. |
| **Are there built‑in team roles and permissions?** | Multi‑user support may be necessary for larger merchants; it is not mentioned in public docs. | Check KNO’s admin UI after installation or contact support to see if it supports inviting team members with different roles. |
| **Does KNO support partial survey saves (incomplete submissions)?** | Understanding whether partial responses are stored affects analytics design and event handling. | Install the app on a dev store and attempt to close the Thank You page mid‑survey; inspect the dashboard to see if partial responses appear. |
| **Are there rate limits on promo code generation?** | Shopify may restrict the number of discount codes created per minute, affecting the design of the promo action【295004260503847†L77-L104】. | Consult Shopify Discounts API documentation and run load tests to determine safe limits. |
| **What is the average response rate on Thank You pages?** | The app claims high response rates but does not disclose numbers; this impacts success metrics and adoption messaging. | Search KNO’s marketing blog or contact existing users to gather empirical response rate data. Alternatively, review case studies if available. |
| **How does KNO handle translations and localisation?** | The app listing mentions support for multiple languages【978605946208732†L102-L123】, but the mechanism (manual translation vs automatic) is unclear. | Explore the builder to see if there is a locale selector or integration with Shopify’s translation API. If not, translation may require duplicating surveys per language. |
| **What data points are sent to integration partners like Triple Whale or Peel?** | The app listing lists many analytics integrations【978605946208732†L121-L123】, but payload details are not documented. | Inspect the integration settings in the app or contact support to obtain field mappings. This will inform how to design our own connectors. |
| **Are there performance or design recommendations for survey placement?** | For example, best practice might be to place the survey at the top of the Thank You page【105345082668391†L146-L151】. Understanding recommended UI patterns helps design default configuration. | Review more documentation or case studies. Ask KNO or merchants about where to place the survey to maximise response rates. |
| **What is the limit on question length or character counts?** | Long prompts might not display well in the checkout extension. | Test with long text in a dev store; check if there is truncation or scrollbars. |
| **How are multiple languages handled in the checkout extension?** | In the checkout extension, translation must come from the merchant; it’s unclear whether the extension provides locale‑specific text automatically. | Evaluate the extension API; test the app in a store set to a non‑English language; see if the survey inherits the locale. |

Addressing these questions will refine the feature set and ensure that the product meets merchant expectations.
